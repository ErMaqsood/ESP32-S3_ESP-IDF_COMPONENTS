Welcome to the I2C peripheral components library with ESP-\/\+IDF support. This library of components is relatively active and updated on a regular basis and a list of operational components and the state of each component under development is available below.

ESP-\/\+IDF I2C device peripheral components supported at this time are listed as follows\+:


\begin{DoxyItemize}
\item ASAIR AHTXX
\item AKM AK8975
\item AMS AS3935
\item OSRAM AS7341
\item ROHM BH1750\+FVI
\item Bosch BME680
\item Bosch BMP280
\item Bosch BMP390
\item Scio\+Sense CCS811
\item Scio\+Sense ENS160
\item Texas Instruments HDC1080
\item Honeywell HMC5883L
\item Texas Instruments INA226
\item Lite-\/\+On LTR390\+UV
\item Maxim-\/\+Integrated MAX30105 -\/ Work in Progress
\item Melexis MLX90614
\item Memsic MMC56\+X3
\item Inven\+Sense MPU6050
\item Sensirion SGP4X
\item Sensirion SHT4X
\item Generic SSD1306 (128x32 and 128x64)
\item Infineon TLV493D -\/ Work in Progress
\item Vishay VEML6040
\item Vishay VEML7700
\end{DoxyItemize}

The above components have a dependency on the {\ttfamily esp\+\_\+type\+\_\+utils} component that must be included in your project. This component includes transmit and receive buffers of various sizes, and byte manipulation features.

Components are developed when needed for product development and/or prototyping purposes, and shared with the ESP32 community for ESP-\/\+IDF developers. Custom component development is available upon request, please contact me directly, and I would be happy to discuss the details of your component needs. If any problems arise please feel free to log an issue and if you would to contribute please contact me.
